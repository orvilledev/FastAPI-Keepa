New Vendor (JFS)

Implementation summary — MSW Overwatch

MetroShoe Warehouse
July 2026 · App v3.0.0
STATUS: SUCCESSFULLY IMPLEMENTED

JFS (Josef Siebel) is now a full vendor in MSW Overwatch — alongside DNK, CLK, OBZ, and the other brands. Staff can manage its MAP and UPCs, schedule its Daily Run, run Express Jobs, and include it in Off-Price Analytics, using the same workflows as every other vendor.

What it does

Adding a new vendor wires Josef Siebel into the Keepa Off-Price system under the short code jfs. Once MAP prices and UPCs are loaded and the Daily Run schedule is set, the brand is monitored on the same cadence and screens as Dansko, Clarks, and the rest — without changing how those existing vendors work.

Business value

  • Josef Siebel monitored with the same Off-Price process as other brands
  • One place for MAP, UPCs, Daily Run, Express Jobs, and Analytics
  • No disruption to DNK, CLK, or other existing vendor setups
  • Staff already trained on Daily Runs can operate JFS immediately

Required before first live run: Update Manage MAP for vendor code jfs, update Manage UPCs for category jfs, then set the Daily Run schedule (time, mode, and email recipients) on the JFS Daily Run page.

How staff finish setup (6 steps)

  1. Confirm JFS (Josef Siebel) appears under Daily Runs, Manage UPCs, and vendor lists (Express Jobs / Analytics).
  2. Update MAP — open Manage MAP, select or enter vendor code jfs, and upload or add Josef Siebel MAP prices.
  3. Update UPCs — open Manage UPCs → JFS (Josef Siebel) and add or import the UPC list for this brand.
  4. Set the Daily Run — open Daily Runs → JFS, choose schedule (time / days), turn the run On, and pick API Mode or Import Mode.
  5. Save email recipients (and BCC if needed) so Off-Price reports go to the right people.
  6. Optional: enable the Dashboard Remind me alert for JFS, and confirm the vendor appears on Off-Price Analytics.

What must be updated

  • MAP — minimum advertised prices for Josef Siebel UPCs, tagged as vendor jfs
  • UPCs — product list under category jfs (defines what each Daily Run / Express Job checks)
  • Daily Run — schedule, mode (API or Import), and recipients must be configured before automated monitoring starts

Where it appears in the app

Area Use for
Daily Runs → JFS Schedule, mode, Same Day Run, reminders, report history
Manage UPCs → JFS Maintain the Josef Siebel UPC list
Manage MAP Load / edit MAP prices with vendor code jfs

Who has access

User Access
Keepa-access users Full JFS Daily Run, Manage UPCs, Manage MAP, Express Jobs, and Analytics (same as other vendors)

Privacy & speed

  • Uses the same secure Keepa and database paths as DNK, CLK, and other vendors
  • No separate login or tool — JFS lives inside MSW Overwatch
  • Adding JFS does not slow or change schedules for other brands

Rollout checklist

  • Confirm JFS card appears on Dashboard / Daily Runs menu
  • Update Manage MAP with Josef Siebel prices (vendor jfs)
  • Update Manage UPCs for JFS (category jfs)
  • Open Daily Runs → JFS and turn the scheduler On
  • Set Daily Run time, days, and API / Import mode
  • Save email recipients for JFS reports
  • Run a short Express Job or Same Day Run to verify MAP + UPCs
  • Confirm JFS shows in Off-Price Analytics (tracking on)
  • Optional: enable T-30 “Remind me” for JFS on the Dashboard
  • Brief the team that Josef Siebel uses code jfs everywhere

Notes for leadership

  • Vendor short code: jfs · Display name: JFS (Josef Siebel)
  • Database seed script: seed_jfs_vendor.sql (scheduler + analytics tracking rows)
  • Until MAP and UPCs are loaded and Daily Run is set, automated Off-Price monitoring for JFS will not produce meaningful results
  • Existing vendors (DNK, CLK, OBZ, REF, BOR, SFF, TEV, CHA) are unchanged
